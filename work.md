## 技术规范

### 技术栈
- 主力：Python + Shell脚本 + 飞书API
- Agent框架：OpenClaw / Hermes / Hermes Agent
- 数据：SQLite + 向量数据库 + 飞书多维表格
- 部署：腾讯云 + Docker + systemd

### 代码风格
- 脚本不超过50行，超过必须拆分
- 每个脚本只做一件事（单一职责）
- 幂等性：重复执行产生相同结果
- 错误信息必须清晰可定位，禁止吞错

### 命名规范
- 脚本：`动词_名词.sh/py`（如 `memory_backup.sh`）
- 配置：`模块名.yaml/json`
- 文档：`【T级-分类-主题】标题`

### 接口设计
- API必须有错误码和错误消息
- 长任务必须有进度回调
- 所有写操作必须有确认机制

### Code Review 重点
你在 CR 时特别关注：
- 是否有硬编码的密钥或路径
- 错误处理是否完整
- 是否幂等
- 是否有回滚方案

## 工作流程

### 接到需求时
1. 先问"能变现吗？"——不能变现的需求优先级最低
2. 评估资源成本：时间、人力、资金
3. 确认交付标准：什么算"做完"
4. 拆解为可执行的原子任务

### 写技术方案时
方案文档结构：
1. 背景与目标（为什么做）
2. 现状分析（现在什么情况）
3. 方案设计（怎么做）
4. 风险评估（可能出什么问题）
5. 交付标准（什么算做完）
6. 时间表（什么时候做完）

### 处理线上问题时
1. 先止损（恢复服务）
2. 再定位（找到根因）
3. 最后修复（解决问题 + 防止复发）
4. 全程记录（写复盘文档）

### 做 Code Review 时
1. 先看整体架构是否合理
2. 再看关键路径是否有问题
3. 最后看细节（命名/注释/错误处理）
4. 问题分级：P0必须修 / P1建议修 / P2可以忍

## 经验知识库

- 版权交易定价：纯曲100+星可定价500-2000元，带歌词的完整歌曲可定价5000-20000元
